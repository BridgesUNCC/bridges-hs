Introduction

Welcome to the high school section of the NSF Bridges project at UNCC! This website is intended to help high school teachers implement Bridges assignments in their classrooms. This article provides a concise overview and explanation of the website.

If you have not already set up Bridges with your IDE of choice, a tutorial is available here.

Note: The high school section of Bridges only uses the Java programming language.

Courses

This website provides instructional materials for the computer sciences courses AP CS A and Advanced Topics.

  • AP CS A: This course consists of assignments corresponding to the AP CS A curriculum as outlined in the AP Computer Science A Course Description.
  • Advanced Topics: This course consists of assignments corresponding to curriculum beyond AP CS A. Many of these topics are also present in first or second year college computer science courses.

When viewing either course, every assignment is tagged with the topic(s) it covers. In addition, a more detailed list of topics can be found within each assignment page.

Assignments

Project-based learning is the foundation of Bridges. Each assignment is designed to teach computer science concepts. An assignment page has the following structure:

  • Description
  • (Optional) Details / concept explanation
  • Topics
  • Intended learning outcomes
  • Instructional materials
  • (Optional) Additional information
  • Documentation

Description

This provides a basic overview of the project. Think of the description as a “pitch” for you to use the assignment.

(Optional) Details / concept explanation

Some assignments require further information or explanation about a specific topic. This section is optional, and most assignments do not include it.

Topics

This is a list of all the topics covered in the assignment. The tags found in the assignment list are not exhaustive, but the topics list is.

Intended learning outcomes

Each assignment has been designed with specific ILOs in mind. It is important to note that the ILOs are written with the assumption that all instructional materials (see below) are used. So, if you adapt the assignment, the ILOs may change.

Instructional materials

This is the most important part of the assignment page. The instructional materials consist of three sections: student, teacher, and resources.

Student

Each assignment includes a student worksheet which consists of instructions, information, and questions for students as they program. The worksheets enable students to work at their own pace, and the questions help you assess student understanding of concepts. Of course, you can adapt these worksheets to meet the needs ofyour classroom.

Additionally, some assignments include test cases or example outputs to help students check their work.

Teacher

Each assignment includes answers keys for the student worksheet. Some of the more challenging assignments also have a teacher walkthrough explaining the code and worksheet. Finally, most assignments include a rubric which you can change if you wish.

Resources

The resource section can vary significantly depending on the project, but it will always include solution files and scaffold files. The scaffold is what you provide to students, and the solution is the finished product. Scaffolds usually include directions or assistance that correspond to the student worksheet.

Additionally, the resources section sometimes includes data files, additional tutorials, and links embedded in the student worksheet.

(Optional) Additional Information

This section includes information such as the source of the project if it is adapted from another resource.

Documentation

The documentation section provides links to all the relevant classes of the assignment. It is typically expected that reading documentation is necessary to complete the assignment.
